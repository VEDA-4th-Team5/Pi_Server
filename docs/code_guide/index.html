5. 센서·주차 상태 흐름

STM32/UART 메시지 (향후 실제 장치 연결)
현재 parking/sensor/hall MQTT 또는 실제 UART/LoRa 입력
→ MqttEventBridge 또는 SensorLinkManager
→ HallParkingService::handleLine()
→ SensorProtocolParser::parse()
→ SensorProtocolMessage
→ ParkingSensorEventAdapter::adapt()
→ SensorSlotIndex에서 sensor_id → slot_id 매핑
→ ParkingSensorSequenceGuard 중복/역순 검사
→ parking::ParkingSlotManager::apply()
→ OCCUPIED: 최신 RTSP ROI 저장 → ACTIVE 세션 → OCR
→ VACANT: 세션 종료 → 위반 전이면 이미지/IMAGE_LOG 삭제
MQTT test, UART line, LoRa CRC frame은 전송 계층만 다르다. 세 입력 모두 검증된 센서 문자열을 같은 HallParkingService::handleLine()에 전달한다.

parking/은 센서 기반 상태 전이 도메인이고, parking_timer/는 EV/PHEV 장기 점유 시간 관리다. 이름이 비슷하지만 책임이 다르다.

6. 장기 점유 타이머 흐름

OCR callback → ParkingSlotManager::handleRecognizedSession(session_id, "EV01", plate)
→ EventDatabase::classifyVehicle()
→ EventDatabase::findActiveBySlot("EV01")
→ 기존 ACTIVE session_id를 그대로 사용
→ TimerManager::schedule()
→ deadline 도달
→ EvidenceProvider가 최신 RTSP ROI 저장
→ EventDatabase::markViolation()
→ VIOLATION_TRIGGERED 이벤트 → Qt MQTT

handleExit("EV01")
→ EventDatabase::departActiveBySlot()
→ 상태 ENDED, exit_time/duration_sec 저장
→ 큐의 과거 노드는 DB 조건부 UPDATE로 무효화
타이머와 DB는 int zone_id가 아니라 std::string slot_id를 사용한다. 따라서 EV01이 숫자 변환 없이 그대로 저장·조회된다.

이미지 binary는 MQTT로 전송하지 않는다. MQTT는 상태와 URL을 전달하고 실제 파일은 HTTP로 pull한다.

8. 최상위 폴더별 역할

폴더역할학습 포인트
include/공개 타입, 클래스 인터페이스, 함수 선언구현 전에 책임과 의존관계를 파악할 때 먼저 읽는다.
src/서버와 타이머의 C/C++ 구현main.cpp에서 시작해 호출되는 모듈을 따라간다.
tests/단위·통합·타이머 테스트 및 fixture클래스의 기대 동작을 가장 빠르게 확인한다.
db/단일 SQLite 스키마와 seed 데이터테이블 관계와 실제 EV01 ID를 확인한다.
config/타이머 및 센서-슬롯 실행 설정DB·제한시간과 HALL01~04 → EV01~04 매핑을 확인한다.
driver/프로젝트 전용 Linux 커널 모듈parking_alert.ko가 /dev/parking_alert를 생성한다.
tools/DB 백업·초기화·조회, 문서 생성 자동화운영 보조 도구이며 서버 런타임 코드는 아니다.
docs/아키텍처, DB, MQTT, HTTP, 테스트 및 자동 API 문서설계 의도와 현재/목표 구현 차이를 확인한다.
data/실행 중 생성되는 DB·Snapshot·BestShot·전처리 파일소스가 아니라 런타임 데이터이며 환경에 따라 없을 수 있다.
CMakeLists.txt서버·테스트·타이머·문서 타깃 정의어떤 소스가 어떤 실행 파일에 링크되는지 확인한다.

12. 학습 권장 순서

  1. src/main.cpp에서 객체 생성과 시작/종료 순서를 읽는다.
  2. include/ 헤더로 각 클래스의 공개 책임만 먼저 익힌다.
  3. MQTT 이벤트 한 건을 기준으로 MqttEventBridge → SnapshotStorage → OcrWorker → EventDatabase를 추적한다.
  4. BestShot 한 건을 기준으로 BestShotReceiver → OcrWorker → DB를 추적한다.
  5. tests/에서 기대 상태 전이를 확인한 뒤 구현 cpp를 읽는다.
  6. 헤더의 Doxygen 주석과 구현 파일에서 private helper와 실제 호출 위치를 확인한다.
문서와 코드가 다르면 코드와 테스트가 기준이다. 구조가 변경되면 이 HTML의 설계 설명을 갱신하고 cmake --build build --target docs로 PDF와 API 문서를 다시 생성한다.
